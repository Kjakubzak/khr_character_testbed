# Getting Started

A step-by-step guide to running the `KHR_character` sample project. For the short version, see the repo [README](../README.md).

## Prerequisites

- **Unity 6000.0.76f1** (the version this project is pinned to). Built-in Render Pipeline by default.
- **Git** with **Git LFS** installed (`git lfs install`) — sample binaries (`.glb`) are tracked via LFS.
- **Internet access on first open** — Unity resolves the UnityGLTF package (including the `KHR_character` plugin) from a pinned Git URL in `Packages/manifest.json`.

## 1. Clone

```bash
git lfs install
git clone <this-repo-url>
cd khr_character_testbed
```

If you cloned before installing LFS, run `git lfs pull` to fetch the real binaries (otherwise `.glb` files are small text pointers).

## 2. Open in Unity

Open the folder with **Unity 6000.0.76f1** (via Unity Hub). On first open Unity will:
- resolve `org.khronos.unitygltf` from the pinned Git commit (this can take a minute),
- import the project and compile the sample assemblies.

## 3. Generate samples + scenes (first time only)

The sample characters and demo scenes are produced by editor tools (so the samples are themselves generated by the exporter under test). If `Assets/SampleAssets/Synthetic/*.glb` or the demo scenes aren't present, run, from the menu bar:

1. **Assets ▸ UnityGLTF ▸ KHR Character ▸ Generate Sample Characters** — writes `SC-Face`, `SC-FacePlus`, `SC-Body` GLBs.
2. **Assets ▸ UnityGLTF ▸ KHR Character ▸ Build Sample Scenes** — generates the 7 scenes and adds them to Build Settings.

The KHR plugins ship enabled in this project; if they ever aren't, run **Assets ▸ UnityGLTF ▸ KHR Character ▸ Enable Plugins**.

## 4. First success (≈1 minute)

1. Open `Assets/Samples/_Shared/Scenes/SampleHub.unity`.
2. Press **Play**.
3. Click **Expressions**.
4. Drag a slider (e.g. `jawOpen`) → the face deforms live.

Use **Back to Hub** to return and try the other demos.

## 5. Where to go next

- Explore the six demos (GLB Viewer, Expressions, Gaze & Camera, Rig & Pose, Round Trip, Health) — see the README table.
- Run the PlayMode tests: **Window ▸ General ▸ Test Runner ▸ PlayMode ▸ Run All**.
- Add your own demo: see [adding-a-demo.md](adding-a-demo.md).
- Continuous integration + validation: see [ci.md](ci.md).

## Neutralize your own character (bring-your-own)

When `Assets/SampleAssets/khr-character-example.glb` is present, **all demo scenes default to that hero character** (consumed via `KHR_character`; `VRMC_*` ignored); otherwise each demo uses its synthetic `SC-*` fallback.

The **Round Trip** demo can also consume

The **Round Trip** demo can also consume an *external* character — e.g. a VRoid/VRM `.glb` that carries `VRMC_*` vendor extensions — purely via `KHR_character`, and re-export it **vendor-neutral**. Drop a file at `Assets/SampleAssets/khr-character-example.glb` (it is **git-ignored**: its mesh/textures carry their own licensing, so keep it local), open **Round Trip** and click **Load + Neutralize external**, then compare the **source** `extensionsUsed` (including `VRMC_*`) against the **re-export** `extensionsUsed` (KHR-only, no vendor extensions, empty `extensionsRequired`). With no such file present it falls back to the neutral `SC-Body`.

## Troubleshooting

| Symptom | Cause / fix |
|---|---|
| **SampleHub shows a red "plugins disabled" banner** | Run **KHR Character ▸ Enable Plugins**. |
| **A demo loads a character with "no KHR Character data"** | The GLB has no `KHR_character` root (a plain glTF) — expected; use an `SC-*` asset. |
| **`SC-*.glb` not found / scenes missing** | Run **Generate Sample Characters**, then **Build Sample Scenes**. |
| **Package didn't resolve / KHR types missing** | First open needs internet + Git to clone the pinned UnityGLTF commit. Check `Packages/manifest.json` / `packages-lock.json` and the Console. |
| **`.glb` files look like tiny text files** | Git LFS wasn't installed before clone — run `git lfs install && git lfs pull`. |
| **Model is pink / unlit** | A render-pipeline shader is missing for your RP. This project targets the Built-in pipeline; ensure the demo shaders are present (URP users: import the URP shader variants). |
| **Rig & Pose stays Generic when switching to Humanoid** | The character's skeleton is missing required humanoid bones — expected graceful fallback; use `SC-Body`. |

> This project tracks the **non-ratified** `KHR_character_*` extensions (glTF PR #2512). Schema and behavior may change.
