# Adding a Demo

This project builds demo scenes **from code** (so they're reproducible and diffable). A demo is a small controller `MonoBehaviour` plus a one-line registration; the scene `.unity` file is generated by `SceneBuilder`. Here's the whole pattern.

## 1. Write a controller

Put it under `Assets/Samples/KhrCharacter/` (assembly `Samples.KhrCharacter`) ŌĆö or a new `Assets/Samples/<Area>/` with its own asmdef referencing `Samples.Shared`.

```csharp
using Samples.Shared;            // CharacterLoader, DemoUiBuilder, OrbitCameraRig, CaveatBanner, BackToHubButton
using UnityEngine;
using UnityGLTF.KhrCharacter;    // KhrCharacter hub + sub-controllers

namespace Samples.Characters
{
    public class MyDemoController : MonoBehaviour
    {
        async void Start()
        {
            // 1. Load a sample by name (committed under Assets/SampleAssets/Synthetic/).
            var root = await CharacterLoader.LoadAsync(
                CharacterLoader.SyntheticPath("SC-Face.glb"), transform);
            if (root == null) return;

            // 2. Frame it, then wire UI only once the hub is ready.
            GetComponent<OrbitCameraRig>()?.Frame(/* bounds */);

            var hub = root.GetComponent<KhrCharacter>();
            if (hub == null) { /* plain glTF: show "no KHR Character data" */ return; }

            hub.WhenReady(c =>
            {
                var ui = gameObject.AddComponent<DemoUiBuilder>();   // or reuse the scene's panel
                var ec = c.Expressions;
                // index-based writes ŌĆö never foreach over Expressions while calling SetWeight
                for (int i = 0; i < ec.Expressions.Count; i++)
                {
                    var h = ec.Expressions[i];
                    ui.AddSlider(h.Name, v => ec.SetWeight(h.Name, v));
                }
            });
        }
    }
}
```

Key shared helpers (all in `Samples.Shared`):
- **`CharacterLoader`** ŌĆö `LoadAsync(absPath, parent)`, `LoadFromBytesAsync(glb, parent)`, `ExportToGlb(root, out GLTFRoot)`, `SyntheticPath(name)`, `EnableImportPlugin(rig)`. It enables the import plugin on the shared settings for you.
- **`DemoUiBuilder`** ŌĆö code-built overlay uGUI: `AddSlider/AddToggle/AddButton/AddDropdown/AddLabel`, `ClearRows`.
- **`ExpressionControlPanel`** ŌĆö drop-in: auto-builds a row per `ExpressionController.Expressions` on `WhenReady`.
- **`OrbitCameraRig`**, **`RenderPipelineBootstrap`**, **`CaveatBanner`**, **`BackToHubButton`**, **`HealthPanel`**.

Rules of thumb: bind through **`KhrCharacter.WhenReady`** (never `Awake` ordering); **null-check** every sub-controller (a capability may be absent); write expression weights **by index**; keep UI **RP-agnostic** (uGUI works on Built-in/URP/HDRP).

## 2. Register it on the hub

Add an entry to `Assets/Samples/_Shared/DemoHubRegistry.cs` so SampleHub shows a launch button:

```csharp
new DemoEntry("MyDemo", "My Demo", "One-line description shown on the hub."),
```

The `SceneName` must match the scene you build in the next step.

## 3. Generate the scene

In `Assets/Editor/SceneBuilder.cs`, add your scene to `BuildAllScenes()` so it's constructed (camera + `OrbitCameraRig`, `RenderPipelineBootstrap`, an `EventSystem`, your controller) and registered in **Build Settings** (required for `SceneManager.LoadScene` by name from the hub). Follow the existing scenes as a template, then run **Assets Ō¢Ė UnityGLTF Ō¢Ė KHR Character Ō¢Ė Build Sample Scenes**.

## 4. Add a test

Add a bounded PlayMode test in `Assets/Tests/` (assembly `Sandbox.Tests`) that exercises your demo's load/drive path ŌĆö it doubles as an anti-hollow gate (it won't compile if the plugin dependency goes missing):

```csharp
[UnityTest]
public IEnumerator MyDemo_DoesTheThing()
{
    var task = CharacterLoader.LoadAsync(CharacterLoader.SyntheticPath("SC-Face.glb"), null);
    while (!task.IsCompleted) yield return null;
    var hub = task.Result.GetComponent<KhrCharacter>();
    Assert.IsNotNull(hub);
    // ... assert your behavior ...
}
```

## 5. Conventions

- **Public-clean:** neutral names/comments only ŌĆö no internal/vendor references; this is a public Khronos-community example.
- **License-clean assets:** if your demo needs a new sample, generate it via `SampleCharacterFactory` (CC0/synthetic) and record it in `Assets/SampleAssets/ATTRIBUTION.md`. No third-party art without a clear open license.
- **Caveats:** add a `CaveatBanner` so the demo never over-promises (e.g. one character per document; `blendMode`/`priority` are runtime-only, not on the wire).

That's it ŌĆö controller + registry entry + SceneBuilder entry + a test.
